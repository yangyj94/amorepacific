클래스 작명 가이드

  • 클래스 작명은 해당 영역이 어떤 영역인지 인지할 수 있도록 작성한다. 의미없는 작명 X
  • 형태-의미 형식으로 작성하는것을 지향하며 상황에 따라 형태-형태, 의미-의미도 가능하나 통일성을 위해 가급적 사용을 지양한다.
    ex) wrap-info(형태-의미) / wrap-btn(형태-형태) / info-detail(의미-의미)
  • 주요 레이아웃의 경우 한 단어만으로 클래스 작명 가능하다.
    ex)header, main 등
  • 영역을 구분하는 클래스명 내에 같은 영역 클래스를 사용하지 않는다.
    ex)div.wrap-xxxx(부모) > span.wrap-nnnn(자식)
