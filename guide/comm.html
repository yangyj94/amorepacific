공통 가이드

1. head

  • 공통으로 연결해야하는 링크는 편리성을 위해 head에 연결한다. 스크립트도 예외 대상은 아니다.
  • 라이브러리 사용시 cdn은 지양한다.

2. Semantic Markup

  • 레이아웃 구성시 시멘틱한 태그를 사용하는것에 중점을 둔다.

3. 웹표준&웹접근성

  • 웹 표준에 맞는 코드를 사용한다.
  • 웹 접근성에 맞게 대체 텍스트를 적절히 제공한다.
  • 웹 접근성에 준수하는 aria속성을 적절히 사용한다.
  • 코드작성후 웹표준 검사기를 돌려 검토한다.

4. 주석

  • 개발 적용이 필요하거나 마크업만으로 구현이 불가능한 경우 주석으로 설명을 제공한다.
  • 주요 영역은 주석으로 분리하여 가독성을 준다.

5. 스프라이트 이미지

  • 트래픽을 낮추기위해 공통으로 사용되는 아이콘 이미지의경우 스프라이트 이미지로 제작하여 사용한다.
  • 반응형시 고정값이 아닌 비율값으로 아이콘이 줄어드는경우 스프라이트 이미지 사용을 지양한다.

6. 프레임워크 환경고려

  • 프레임워크 환경을 고려하여 css적용시 인접선택자 및 순서선택자 사용에 주의한다.
